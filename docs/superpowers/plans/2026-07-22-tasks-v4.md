# Tasks Tab v4 (W4) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `general` task type with a preset library aligned to the expansion roadmap, board filtering/grouping that keeps a packed board readable, and restore the dormant weekly-history export.

**Architecture:** The server type/checklist machinery generalizes minimally (one new type, optional checklist). Presets are a code-defined library consumed by the creation modal. Filtering and per-column type grouping are pure client-side concerns layered onto the existing board.

**Tech Stack:** Node 20 / Express / MongoDB / vitest (server), React 19 / framer-motion / TanStack Query / vitest (frontend).

**Spec:** `docs/superpowers/specs/2026-07-22-tasks-v4-design.md`
